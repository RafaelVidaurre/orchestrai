# Stori Product Spec v1

Status: Board-aligned product direction synthesized from board questionnaire answers on March 13, 2026.

## Executive Direction

- Build Stori first for serious indie game developers and small game teams.
- Ship a single-creator v1 while keeping the architecture ready for future team collaboration.
- Enter the workflow at blank-slate worldbuilding with a guided, skippable setup experience.
- Win on exceptional UX, opinionated structure, and trustworthy canon management.
- Ship a single responsive web product that works on desktop and mobile web.
- Keep AI out of the initial release, but leave room for future AI-assisted linking, consistency checks, and guided assistance.

## Product Thesis

Stori should be the craft-first worldbuilding system for game creators who need more than a notes app and less chaos than a custom wiki. The wedge is not generic AI generation. The wedge is a polished product that helps creators start a world correctly, structure it with confidence, and trust the canon as it grows.

## Launch Customer

Serious indie game developers and small game teams.

## Core User Problem

Current worldbuilding tools make creators do too much manual organizational work. The biggest failures are:

- poor UX that makes worldbuilding feel like admin work
- weak structure that leaves users guessing how to organize their world
- difficulty keeping canon linked, updated, and searchable
- growing mistrust in the source of truth as the world becomes larger

## Core Job To Be Done

Help a creator go from a blank page to a structured, cross-referenced world bible they can trust and expand over time.

That means Stori must:

- guide first-time world setup
- preserve consistency as the world grows
- make canon fast to navigate and retrieve later

## Product Principles

- Craft first
- Opinionated structure
- Trustworthy canon
- UX is strategic
- AI must respect authorship

## v1 Scope

- Guided onboarding wizard for blank-slate ideation and world setup
- Opinionated entity model for:
  - characters
  - locations
  - factions
  - power systems or rules
  - cultures
  - items
  - timeline elements
- Fast structured entity creation
- Linking and navigation across canon relationships
- Search and browsing for retrieval
- Responsive desktop and mobile web
- Single creator first

## Explicit Non-Goals

- AI-first positioning
- autonomous AI writing
- collaboration-first scope
- native desktop or mobile apps
- broad integrations or migration tooling at launch
- full writing-suite or publishing workflow depth

## Quality Bar

The first release should favor fewer features with stronger:

- visual design
- reliability and data integrity
- ease of use

## Execution Sequence

1. Guided Foundation
2. Linked Canon
3. Monetization and Expansion Hooks
