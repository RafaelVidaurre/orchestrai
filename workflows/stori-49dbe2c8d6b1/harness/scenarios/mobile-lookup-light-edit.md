# Mobile Lookup And Light Edit

## Why This Scenario Exists

Mobile is not the primary creation surface, but it must still support the core promise of trustworthy canon on the go.

## Representative User

A creator is away from their desk, needs to confirm a world fact, and makes a quick correction or link adjustment from mobile web.

## Task

Open the responsive mobile web experience, find a known entity, inspect its linked canon, and make a small edit without needing desktop-only affordances.

## Expected Outcome

- navigation and search remain usable on a narrow viewport
- entity pages preserve hierarchy and link clarity on mobile
- small edits such as summary fixes or link adjustments are possible
- the experience stays focused on lookup and maintenance, not deep long-form authoring

## Validation Ideas

- run the same lookup flow on a narrow viewport
- verify touch targets and relationship sections remain legible
- make a short edit and confirm it persists

## Failure Signals

- mobile hides the most important canon context
- basic lookup requires desktop-only controls
- light edits are technically possible but frustrating enough to be avoided
