# Search And Retrieval

## Why This Scenario Exists

Stori's value compounds only if creators can retrieve trusted facts quickly after the world becomes large.

## Representative User

A creator remembers a detail about a rule, event, or location but not the exact title and needs to recover it quickly.

## Task

Use search or browse affordances to find a canon fact and reopen the source entity with enough surrounding context to trust the answer.

## Expected Outcome

- important entities can be found from imperfect recall
- search results expose enough metadata to distinguish similar results
- reopening a result lands the user in the source-of-truth entity, not an orphaned snippet
- the user can continue browsing from the result into related canon

## Validation Ideas

- search by exact title, partial title, and remembered concept
- verify result ranking for likely queries
- confirm the opened entity exposes nearby structured facts and links

## Failure Signals

- search only works for exact names
- users land on stripped-down previews instead of trusted entity pages
- similar results are impossible to distinguish
